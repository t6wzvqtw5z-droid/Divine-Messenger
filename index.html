1. Техническое резюме (Technical Summary)

Протокол и Шифрование

Мессенджер Divine разворачивается на базе кастомной реализации бинарного протокола MTProto 2.0 поверх TCP/Веб-сокетов, оптимизированного под низкую задержку (low latency). Облачные чаты защищены сквозным шифрованием на участке клиент-сервер, секретные чаты и звонки 1-1 используют чистый End-to-End (E2EE) Diffie-Hellman с генерацией ключей на устройствах пользователей.

Стек технологий Бэкенда

  • Язык разработки: Go (Golang) — высокая производительность конкурентных соединений (Goroutines).
  • Слой хранения (Основная БД): PostgreSQL + TimescaleDB (для горизонтально масштабируемой и быстрой истории сообщений).
  • Слой кэширования и очередей: Redis Cluster (хранение активных пользовательских сессий, кеширование состояний "онлайн", брокер очередей сообщений).
  • Распределенное хранилище файлов: MinIO (S3-совместимое) для медиафайлов, стикеров и голосовых сообщений, прикрытое CDN.

Real-time звонки и обход NAT

Голосовые и видеовызовы реализуются через стек WebRTC. Для обеспечения коннективности между клиентами в любых сетях применяется архитектура ICE (Interactive Connectivity Establishment):

  • STUN серверы: Для определения публичных IP-адресов клиентов при прямом соединении (P2P).
  • TURN серверы (Coturn): Если прямое P2P соединение невозможно (симметричный NAT), трафик ретранслируется через распределенные медиа-серверы с минимальной задержкой.
  • Групповые звонки (до 30 человек): Реализуются через SFU (Selective Forwarding Unit) медиа-сервер на базе LiveKit или Janus. Клиент отправляет один поток, сервер раздает его остальным участникам, снижая нагрузку на исходящий канал спикера.

Авторизация и Синхронизация

Регистрация базируется строго на телефонном номере. При входе генерируется уникальный auth_key устройства по протоколу DH. Если аккаунт активен на другом девайсе, RPC-запрос отправляет системный верификационный код во внутренний сервисный чат Divine. SMS шлюз (Twilio/Infobip) задействуется как fallback-метод.

2. Пошаговый план разработки (Sprint: 2 недели, 2 Dev + 1 QA)

Ресурсная сетка: Dev 1 (Backend/Инфраструктура), Dev 2 (Frontend/Mobile - Flutter мультиплатформа), QA (Тестирование/Автотесты).

Дни Этап / Задачи Ответственный Результат (Milestone)
День 1-3 Развертывание инфраструктуры. Настройка БД, Redis, Coturn. Генерация базовых схем MTProto. Реализация слоя авторизации (sendCode, signIn). Dev 1, Dev 2 Готов бэкенд авторизации, базовое мобильное приложение открывает экран ввода телефона.
День 4-6 Реализация движка облачных сообщений (транспорт через WebSockets). Синхронизация состояний чатов при входе. Локальная БД клиента (SQLite). Dev 1, Dev 2 Работают текстовые сообщения 1-1, статусы доставки (одна/две галочки), форматирование текста.
День 7-9 Интеграция медиа-сервера (MinIO/S3). Логика групповых чатов. Реализация WebRTC пайплайна для звонков 1-1 через STUN/TURN. Логика профилей и usernames. Dev 1, Dev 2, QA Передаются фото/видео, работают звонки между двумя устройствами в локальной сети и за NAT. QA начинает смоук-тесты.
День 10-12 Доработка UI/UX по гайдлайнам (вкладки, кастомные темы, градиентный фон). Внедрение push-уведомлений. Оптимизация производительности (энергоэффективность). Dev 2, QA Приложение визуально идентично Telegram. Настроены пересылки, ответы, реакции и профили. Полное регрессионное тестирование.
День 13-14 Исправление критических багов (Bugfixing). Сборка релизных билдов под Android (APK) и iOS (TestFlight). Финальный нагрузочный тест бэкенда. Все роли MVP Divine полностью готов к закрытому бета-тесту. Развернут Web-клиент.

4. Инструкция по развертыванию и сборке клиентов

Шаг 1: Клонирование репозиториев

# Клонирование монорепозитория Divine (Backend + Спецификации + Клиенты)
git clone https://github.com/divine-messenger/divine-core.git
cd divine-core

Шаг 2: Инициализация окружения бэкенда

# Копирование и настройка переменных окружения
cp .env.example .env

# Запуск инфраструктуры (PostgreSQL, Redis, Coturn, MinIO) через Docker Compose
docker-compose up -d

# Сборка и запуск основного Go-сервера
go build -o divine-server cmd/server/main.go
./divine-server

Шаг 3: Сборка Мобильного клиента (Flutter-платформа)

Перейдите в директорию клиента и соберите приложения для целевых мобильных ОС:

cd apps/mobile_client
flutter pub get

# Компиляция под Android (Сборка релизного APK)
flutter build apk --release

# Компиляция под iOS (Создание архива для App Store / TestFlight)
flutter build ipa --release

Айдентика и Логотип

Логотип «Divine»: Стилизованное минималистичное белое крыло, вписанное в идеальный круг с небесно-голубым градиентом. Геометрия линий в точности повторяет углы наклона бумажного самолетика Telegram, сохраняя преемственность скорости и легкости.

Дефолтное оформление чатов

  • Задний фон по умолчанию: Мягкий бесшовный градиент от небесно-голубого (#b3e5fc) к нежной лаванде (#e1bee7) с едва заметным полупрозрачным паттерном из геометрических линий (интенсивность паттерна 4%).
  • Расположение элементов:
    • Нижний бар навигации (Mobile): 4 вкладки слева направо — Контакты (иконка силуэта), Звонки (телефонная трубка), Чаты (бабл чата со счетчиком непрочитанных), Настройки (шестеренка с миниатюрой аватара).
    • Список чатов: Аватар слева (круглый), имя и превью сообщения по центру, время и статус доставки (две галочки) — справа. Нажатие на чат открывает экран переписки с сохранением нативного эффекта плавного скольжения (Slide Transition).
